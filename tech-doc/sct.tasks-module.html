read_expt_data(neutron_files, neutron_unit, xray_files, xray_unit, output_path, param)

source code 

Read in lists of neutron and xray experimental data files. Create directories to hold theoretical models and curves used to compare against the experimental data.

Returns the experimental data (converted in to Angstrom if required and including Rg and Rxs1 values).

Parameters:
  • neutron_files (list) - List of files containing Q, I(Q) data from neutron experiments.
  • neutron_unit (string) - Length unit used in neutron data ('a' or 'nm')
  • xray_files (list) - List of files containing Q, I(Q) data from xray experiments.
  • xray_unit (string) - Length unit used in xray data ('a' or 'nm')
  • output_path (string) - Path in which to create output directories
  • param (dictionary) - Dictionary containing standard SCT parameters
Returns: list, list, dictionary
Two lists containing dictionaries of containing the experimental data ('data'), Rg ('rg'), Rxs1 ('rxs') and filename ('file'). One dictionary containing the paths created for the neutron models and data ('dry_model' and 'scn') and the same for xrays ('wet_model', 'scx').

analyse_sphere_model(model, expt_curves, sphere_radius, param, chi2=False, neutron=False)

source code 

Take a sphere model and calculate the theoretical scattering curve. The Rg and Rxs1 are also calculated from the curve and returned alongside the rfac comparing the theoretical curve to an experimental one.

Parameters:
  • model (list) - List of lists containing x, y & z coordinates (3 * floats) for each sphere in a sphere model.
  • expt_curves (list) - List of two dimensional numpy arrays containing scattered vector magnitudes, q, and intensities, I, from experiment.
  • sphere_radius (float) - Sphere radius
  • param (dictionary) - Dictionary containing parameters to use when creating models and analysing curves.
  • neutron (boolean) - Flag set if aiming to create a theoretical neutron curve
  • chi2 (boolean) - Flag set if using Chi^2 to compare curves
Returns: dictionary
Dictionary containing the following key/value pairs:
  • model_rg: Radius of gyration calculated directly from sphere model.
  • curve_rg: Radius of gyration calculated from the theoretical scattering curve derived from the sphere model.
  • curve_rxs1: Cross-section calculated from the theoretical scattering curve derived from the sphere model. If an rxs2 range is provided in param then a 'curve_rxs2' will also be returned.
  • rfac: R factor comparing experimental and theoretical scattering curves.

sas_model_summary_output(theor_data, param)

source code 

Format summary data from modelling for output. Columns are Rg from model, Rg from curve, Rxs1 from curve and volume of sphere model. An Rxs2 from curve column will also be produced if a 'rxs2' fit range is specified in param.

Parameters:
  • theor_data (dictionary, dictionary) - Dictionaries containing the following key/value pairs for the dry model/neutron and wet model/xray comparisons:

    model_rg: Radius of gyration calculated directly from sphere model.

    curve_rg: Radius of gyration calculated from the theoretical scattering curve derived from the sphere model.

    curve_rxs1: Cross-section calculated from the theoretical scattering curve derived from the sphere model. May also contain curve_rxs2.

    rfac: R factor comparing experimental and theoretical scattering curves.

    volume: Sphere model volume

  • param (dictionary) - Dictionary containing parameters to use when creating models and analysing curves.
Returns: string
String containing the input for the sphere model Rg, curve Rg, curve Rxs1 (possibly also Rxs2) and volume

create_comparison_header_text(data_files, metric)

source code 

Create the spacing for needed to provide two columns (Rfactor/Chi^2 and scale) for each input experimental curve in the summary output header. The header has the following format (in the final version it is tab separated) except without the line numbering:

0 Path to input PDBs

1 Neutron X-ray

2 Model Rg_model Rg_curve Rxs1_curve Volume (Rfactor scale) * neutron curves Rg_model Rg_curve Rxs1_curve Volume (Rfactor scale) * xray curves

This function provides the spacing for one set of curves (either x-ray or neutron).

Parameters:
  • data_files (list) - List of input files names containing experimental scattering curves
Returns: list
List of strings containing the spacing and column headings required for the three lines described above.

write_summary_header(in_pdb, in_neut, in_xray, param, output, chi2=False)

source code 

Print the header for the summary data from sphere model analysis. When properties of the sphere model are references dry models are associated with neutrons and hydrated (wet) ones with x-rays. The header has the following format (in the final version it is tab separated) except without the line numbering (also a Rxs2 cloumn is added after Rxs1 if a range is supplied in param):

0 Path to input PDBs

1 Neutron X-ray

2 Model Rg_model Rg_curve Rxs1_curve Volume (Rfactor scale) * neutron curves Rg_model Rg_curve Rxs1_curve Volume (Rfactor scale) * xray curves

Each experimental curve input requires two columns - Rfactor and scale Where the latter is the scaling factor used to match the theoretical and experimental curves in the Rfactor calculation.

Parameters:
  • in_pdb (string) - Path to directory containg input PDB files
  • in_neut (list) - List of input experimental data files (neutron)
  • in_xray (list) - List of input experimental data files (xray)
  • param (dictionary) - Dictionary containing parameters to use when creating models and analysing curves.
  • output (file) - File to use to print output
  • chi2 (boolean) - Use Chi^2 instead of R factor as curve comparison metric
